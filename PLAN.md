# План развития XeeLite

## Текущее состояние

XeeLite — минимальный просмотрщик изображений на SwiftUI (macOS 14+).
Сейчас умеет: открыть файл, собрать список изображений из папки, листать Prev/Next (кнопки и стрелки).

Цель: реализовать функциональность, сопоставимую с Xee³.

---

## Фаза 1 — Базовые улучшения просмотра

Минимальные доработки, которые сразу заметно улучшат повседневное использование.

### 1.1 Zoom (масштабирование)

**Что сделать:**
- Масштабирование колесом мыши / трекпадом (pinch-to-zoom)
- Перетаскивание изображения при зуме (pan)
- Режимы: Fit in Window, Fit on Screen, Actual Size (1:1), произвольный %
- Горячие клавиши: `⌘0` (Fit), `⌘1` (Actual Size), `⌘+`/`⌘-`
- Отображение текущего масштаба в статус-баре

**Как реализовать:**
- `MagnificationGesture` + `DragGesture` на изображении
- `@State` переменные для scale и offset
- Меню View с командами масштабирования

**Сложность:** средняя

### 1.2 Full Screen

**Что сделать:**
- Полноэкранный режим по `⌘F` или двойному клику
- Скрытие всех контролов в полноэкранном режиме
- Показ контролов при движении мыши (auto-hide)

**Как реализовать:**
- `NSWindow.toggleFullScreen()` через WindowAccessor
- Отслеживание `NSWindow.StyleMask.fullScreen`
- Таймер для auto-hide курсора и контролов

**Сложность:** низкая

### 1.3 Расширенная навигация клавиатурой

**Что сделать:**
- `Space` — следующее изображение
- `Backspace` — предыдущее
- `Home` / `End` — первое / последнее
- `⌘←` / `⌘→` — перемотка на 10 изображений

**Как реализовать:**
- Расширить `KeyAwareView.keyDown(with:)`

**Сложность:** низкая

## Фаза 2 — Информация и метаданные

### 2.1 Status Bar (строка состояния)

**Что сделать:**
- Нижняя панель с информацией: имя файла, размеры (px), размер файла, формат, позиция в папке (N/M)
- Возможность скрыть/показать (`⌘/`)

**Как реализовать:**
- Новый компонент `StatusBarView`
- Данные из `NSImage.representations`, `FileManager.attributesOfItem`
- `@AppStorage` для сохранения видимости

**Сложность:** низкая

### 2.2 Панель метаданных (EXIF/IPTC/XMP)

**Что сделать:**
- Боковая панель (Inspector) с метаданными: EXIF (камера, выдержка, ISO, фокусное), IPTC (описание, автор), XMP, GPS-координаты
- Разворачиваемые секции по группам
- Копирование значений по клику
- Переключение по `⌘I`

**Как реализовать:**
- `CGImageSourceCopyPropertiesAtIndex` для чтения метаданных — это нативный API, не нужны внешние зависимости
- Словари `kCGImagePropertyExifDictionary`, `kCGImagePropertyIPTCDictionary`, `kCGImagePropertyGPSDictionary` и т.д.
- SwiftUI `List` с `DisclosureGroup` для секций

**Сложность:** средняя

## Фаза 3 — Анимация и расширенные форматы

### 3.1 Анимированные GIF / APNG

**Что сделать:**
- Воспроизведение анимированных GIF и APNG
- Контролы: play/pause, покадровое переключение, скорость
- Показ номера кадра и общего числа кадров

**Как реализовать:**
- `CGImageSource` для извлечения кадров и длительностей (`kCGImagePropertyGIFDelayTime`)
- Кастомный `NSViewRepresentable` с таймером для анимации
- Или `NSImageView` с `animates = true` через NSViewRepresentable

**Сложность:** средняя

### 3.2 Расширенная поддержка форматов

**Что сделать:**
- Убедиться, что работают все форматы, поддерживаемые macOS нативно: JPEG, PNG, GIF, TIFF, BMP, PSD, HEIC, HEIF, WebP, JPEG 2000, EXR, ICNS, ICO, TGA, PBM/PGM/PPM, RAW (CR2, NEF, ARW, DNG и др.)
- Добавить форматы в `supportedExtensions` и в `Info.plist` при упаковке в .app
- Корректно указать `CFBundleDocumentTypes` и `UTI` для каждого формата

**Что поддерживается из коробки через NSImage/CGImageSource:**
- JPEG, PNG, GIF, TIFF, BMP, PSD (flattened), HEIC, WebP, JPEG 2000, EXR, ICNS, ICO, TGA, RAW (через ImageIO), PBM/PGM/PPM
- Это покрывает ~90% форматов Xee³

**Что НЕ поддерживается нативно (потребуются сторонние библиотеки):**
- Amiga IFF/ILBM, MacPaint, FlashPix, SGI RGB, XBM/XPM, Radiance HDR — экзотические форматы, использование крайне редкое

**Рекомендация:** ограничиться нативными форматами macOS — их более чем достаточно.

**Сложность:** низкая (для нативных), высокая (для экзотических — не рекомендуется)

## Фаза 4 — Управление файлами

### 4.1 Переименование

**Что сделать:**
- Переименование текущего файла по `⌘R` или Enter
- Inline-редактирование в заголовке окна или модальный диалог
- Валидация имени (запрещённые символы, дубликаты)

**Как реализовать:**
- `FileManager.default.moveItem(at:to:)` для переименования
- Обновление `imageURLs` после переименования

**Сложность:** низкая

### 4.2 Копирование / перемещение в папки

**Что сделать:**
- Быстрые действия: скопировать/переместить файл в одну из предустановленных папок
- Настраиваемый список папок-назначений
- Горячие клавиши для каждой папки (1-9)

**Как реализовать:**
- `FileManager.default.copyItem` / `moveItem`
- `@AppStorage` или JSON-файл для хранения списка папок
- Панель настроек для управления списком

**Сложность:** средняя

### 4.3 Удаление файла

**Что сделать:**
- Перемещение в Корзину по `⌘⌫`
- Подтверждение перед удалением
- Автопереход к следующему изображению после удаления

**Как реализовать:**
- `FileManager.default.trashItem(at:resultingItemURL:)`
- Обновление массива `imageURLs` и `currentIndex`

**Сложность:** низкая

### 4.4 Цветовые метки Finder

**Что сделать:**
- Установка цветовой метки файлу (как в Finder: красный, оранжевый и т.д.)
- Контекстное меню или горячие клавиши

**Как реализовать:**
- `URL.setResourceValues()` с `URLResourceValues.labelNumber`

**Сложность:** низкая

## Фаза 5 — Slideshow и продвинутый просмотр

### 5.1 Slideshow

**Что сделать:**
- Автоматическое переключение изображений с настраиваемым интервалом
- Полноэкранный режим слайдшоу
- Контролы: play/pause, выбор интервала
- Переходы между слайдами (fade, slide)

**Как реализовать:**
- `Timer.publish` для автопереключения
- SwiftUI `.transition()` и `.animation()` для переходов
- Комбинация с полноэкранным режимом из 1.2

**Сложность:** средняя

### 5.2 Crop (обрезка)

**Что сделать:**
- Инструмент обрезки: выделение прямоугольной области
- Отображение размеров выделения
- Фиксированные пропорции (свободная, 1:1, 4:3, 16:9)
- Сохранение обрезанного изображения (Save / Save As)

**Как реализовать:**
- Кастомный оверлей с `DragGesture` для выделения области
- `CGImage.cropping(to:)` для обрезки
- `CGImageDestination` для сохранения в нужном формате

**Сложность:** высокая (интерактивный UI обрезки)

### 5.3 Цветокоррекция

**Что сделать:**
- Регулировка яркости, контраста, гаммы
- Ползунки в боковой панели или оверлее
- Применение только к отображению (не деструктивно)

**Как реализовать:**
- CoreImage фильтры: `CIColorControls` (яркость, контраст, насыщенность), `CIGammaAdjust`
- Применение `CIFilter` к `CIImage`, конвертация обратно в `NSImage`
- Можно добавить опцию "сохранить с коррекцией"

**Сложность:** средняя

## Фаза 6 — Многооконность и навигация

### 6.1 Миниатюры (Thumbnail Strip)

**Что сделать:**
- Горизонтальная полоса миниатюр внизу или сбоку
- Выделение текущего изображения
- Клик по миниатюре — переход к изображению
- Ленивая загрузка миниатюр

**Как реализовать:**
- `ScrollView(.horizontal)` + `LazyHStack`
- Асинхронная генерация миниатюр через `CGImageSourceCreateThumbnailAtIndex`
- `NSCache` для кэширования

**Сложность:** средняя

### 6.2 Browser (окно обзора папки)

**Что сделать:**
- Отдельное окно / режим с сеткой миниатюр всей папки
- Сортировка: по имени, дате, размеру
- Фильтрация по формату
- Двойной клик — открыть в режиме просмотра

**Как реализовать:**
- Второй `WindowGroup` или `Window` в SwiftUI
- `LazyVGrid` с миниатюрами
- `FileManager` + `URLResourceValues` для метаданных сортировки

**Сложность:** высокая

### 6.3 Вкладки

**Что сделать:**
- Несколько изображений / папок в одном окне через вкладки
- Нативные macOS-вкладки (`⌘T` — новая, `⌘W` — закрыть)

**Как реализовать:**
- Нативная поддержка вкладок macOS: `NSWindow.tabbingMode = .preferred`
- Каждая вкладка — отдельный `WindowGroup`

**Сложность:** средняя

- Save As / Export с выбором формата (JPEG, PNG, TIFF, HEIC, WebP)

### 7.4 Установить как обои рабочего стола

**Сложность:** низкая

- Автоматическое обновление списка при добавлении/удалении файлов в папке

## Фаза 9 — Поддержка архивов

### 9.1 Просмотр изображений из архивов

**Что сделать:**
- Открытие ZIP, RAR, 7z, TAR и просмотр изображений внутри без распаковки
- Ввод пароля для защищённых архивов

**Как реализовать:**
- Интеграция библиотеки для работы с архивами. Варианты:
  - [ZIPFoundation](https://github.com/weichsel/ZIPFoundation) — Swift, только ZIP
  - [SWCompression](https://github.com/tsolomko/SWCompression) — Swift, ZIP/TAR/GZ/BZ2/7z/XZ
  - Или обёртка над `libarchive` (есть в macOS)
- Создание `ArchiveImageSource` по аналогии с `XeeArchiveImageSource`

**Можем ли реализовать:** да, но это добавляет зависимость в Package.swift.

**Сложность:** высокая

## Что мы НЕ реализуем

| Фича Xee³ | Причина отказа |
|---|---|
| OpenGL-рендеринг | Устаревший подход. SwiftUI + CoreImage достаточно для просмотрщика. Metal был бы оправдан только для огромных изображений (100+ мегапикселей) |
| Экзотические форматы (Amiga IFF, MacPaint, FlashPix, SGI, XBM/XPM) | Крайне редкое использование, нет нативной поддержки — трудозатраты несопоставимы с пользой |
| Paddle / лицензирование | Коммерческая инфраструктура, не относится к функциональности просмотрщика |

## Рекомендуемый порядок реализации

```
Фаза 1  →  Zoom, Full Screen, клавиатура         (фундамент просмотра)
Фаза 2  →  Status Bar, метаданные                 (информативность)
Фаза 3  →  Анимация GIF, расширение форматов      (охват контента)
Фаза 4  →  Переименование, копирование, удаление  (управление файлами)
Фаза 7  →  Буфер обмена, drag & drop, экспорт     (интеграция с системой)
Фаза 5  →  Slideshow, crop, цветокоррекция        (продвинутый просмотр)
Фаза 6  →  Миниатюры, browser, вкладки            (навигация)
Фаза 8  →  Темы, настройки                        (кастомизация)
Фаза 9  →  Архивы                                  (дополнительно)
```

Фазы 7 и 4 поставлены раньше 5 и 6, потому что копирование/вставка и управление файлами проще в реализации и дают ежедневную пользу, тогда как browser и crop — более трудоёмкие фичи.
